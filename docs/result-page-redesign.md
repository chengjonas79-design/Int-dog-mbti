# 结果页精修版改版说明（分享率优先）

## 1. 项目目标
- 第一目标：提升分享率（用户完成测试后愿意主动晒图/转发）。
- 结果页定位：提高“获得感 + 可分享感”，不做解锁机制。
- 场景优先级：微信内 H5，手机竖屏优先。

## 2. 视觉方向
- 风格：温暖治愈，小红书风格（轻暖色调、柔和卡片、截图友好）。
- 气质关键词：亲和、轻松、真实、有内容。
- 设计原则：
  - 一屏内必须出现“核心结论 + 关键证据”。
  - 每个模块都可独立截图分享。
  - 文案避免运营感，不向用户暴露“裂变”等业务术语。

## 3. 页面结构（无解锁）
1. 结果总览卡
2. 情绪泡泡（6 个）
3. 维度条形图（4 条：EI / SN / TF / JP）
4. 毛孩子性格画像
5. 主人相处指南
6. 分享海报（可点头像直接更换，自动刷新海报）

## 4. 文案策略
- 风格：温暖但一针见血，避免空泛夸赞。
- 每块文案保持“短句 + 可执行建议”。
- 结果说明采用：
  - 一句“核心判断”
  - 一句“行为表现”
  - 一句“主人沟通方式”

## 5. 交互策略
- 分享海报区支持：点头像直接更换。
- 换头像后：自动刷新海报预览，不要求用户额外点击更新按钮。
- 维持当前保存逻辑：生成后可长按保存并分享。

## 6. 技术实现要点
- 基于现有 `index.html` 单页结构实现，不引入新框架。
- 使用 `score` 原始分值生成维度百分比，提升“结果可信度”。
- 使用 MBTI 类型字母映射生成画像/指南文案，避免 16 型手写冗余。
- 模块化函数：
  - `buildEmotionBubbles(type, r, score)`
  - `buildDimensionRows(score)`
  - `buildPersonaSections(type, r, score)`
  - `buildGuideSections(type)`

## 7. 验收标准
- 结果页信息密度明显提升，但阅读不拥挤。
- 视觉风格统一（总览卡、图表卡、分享卡一致）。
- 海报分享链路无中断：生成、换头像、自动刷新均可用。
- 手机竖屏下模块顺序清晰，文本不溢出，点击区域有效。
